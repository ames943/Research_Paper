# Phase 2 Simulation — Interpretive Notes

## Signal f²=0.007 is the null model (δ=0)

During calibration, the target signal R²=0.007 was reached at δ=0 (no response
shift applied). This is because the LedoitWolf-fitted multivariate Gaussian
already produces a PERMANOVA response R²≈0.0085 at n=118 through random
variation alone — meaning f²=0.007 is below the sampling-noise floor of the
generator at this sample size.

**This is not a degenerate or missing simulation result.** It directly mirrors
the real data finding: the n=118 three-cohort pooled analysis produced response
R²=0.007, p=0.847 — a result statistically indistinguishable from the null.
The simulation calibration independently recovers the same conclusion: at
n≈118, R²=0.007 is consistent with zero biological signal.

Practical consequence for the heatmaps:
- The sig_f2=0.007 row represents the **null model** (δ=0, no injected signal).
- Its ceiling AUC is ~0.50 by construction (no signal → chance-level prediction).
- Δ AUC = (method AUC) − (ceiling AUC) at sig_f2=0.007 reflects only how
  each correction method handles the regime of pure batch noise — no signal
  to recover or destroy.
- Any apparent non-zero Δ AUC in this row is sampling variance around zero.

The "intrinsic" signal estimate (0.027) uses the n=39 single-cohort result as

"measured" signal (0.007) is what PERMANOVA observes after pooling three
heterogeneous cohorts — batch variance dominates and the response signal
appears to vanish. Both points are marked on the n_per_cohort=40 facets of
the heatmaps.

## Ceiling AUC interpretation

Ceiling AUC is defined as: method="none", n_cohorts=1, batch_f2=0, same
(sig_f2, n_per_cohort). It represents the best achievable AUC with no batch
effects at the given per-cohort sample size and signal strength. Δ AUC < 0
means the method (or multi-cohort pooling itself) is harming performance
relative to this single-cohort reference. Δ AUC > 0 would mean a correction
method extracts more signal than the ideal single-cohort scenario — very
unlikely for any real correction method.
